Overview

In this hands-on lab you'll learn how to assess a customer's existing environment and import data collected from Azure/AWS infrastructure. You'll generate inventory, Performance and Network Dependencies reports as well as financial reports such as TCO and Licenses reports.

What you'll do

  • Collect data automatically from an existing environment
  • Review data in Migration Center
  • Generate various reports
Note: If you're using a Chromebook or Mac, you will need to use a third-party RDP tool, like Spark View, rather than the RDP button in the Cloud Console.

Setup and requirements

Read these instructions. Labs are timed and you cannot pause them. The timer, which starts when you click Start Lab, shows how long Google Cloud resources are made available to you.

This hands-on lab lets you do the lab activities in a real cloud environment, not in a simulation or demo environment. It does so by giving you new, temporary credentials you use to sign in and access Google Cloud for the duration of the lab.

  2. After login is done and GCP console opened, from the Navigation menu, click View All Products at the bottom of the panel, to open a list of all products.

Task 5. Import AWS data

  1. In the windows-vm, using Chrome navigate to the following URL:
https://storage.googleapis.com/spls/gsp1095/vm-aws-import-files.zip

This will start downloading of the file generated from the AWS account.

  1. Decompress the downloaded zip file by right-clicking on it and selecting Extract ALL....

  2. Click Extract.

  3. In Google Cloud, from the Navigation menu, select Migration Center and click on Add Data button.

  4. Select Manual upload option.

  1. Enter aws-account-import into File import job name field.

  2. From File format dropdown select AWS (CSV).

  1. Click Select files to upload button.

  2. Navigate to the directory where the zip file was downloaded to (Default location: C:\Users\migrationcenter\Downloads\vm-aws-import-files).

  3. Select all 4 files in the directory, click Open button. This will bring all selected files to the import page.

  1. Click Upload Files.

  2. Once upload completes there should be a green check next to every file.

  1. Click the Import Data button to finish the import.

  1. Click Confirm on the confirmation popup.
Verify AWS file was imported successfully.

Task 6. Review data in Migration Center

  1. In Google Cloud console, from the Navigation menu, select Migration Center and click on Assets sub-menu.

  1. View list of collected assets.

  2. Click on the VM Name link of one of the assets in the list to view asset details.

  3. Switch between different tabs inside the asset details page to view more assets details such as Insights, Source VM Details, Metadata and Performance data.

Note: Performance data is not available for assets created using the provided AWS import data file.

  1. Click the Back button to get back to the Assets list view.

  2. Select one or more assets to bring up the additional options like Select all button, for all available assets selection, based on applied filter.

  1. Deselect previously selected assets and select only few assets. Bring them out of scope by clicking the Out of scope button.

  1. Provide any relevant reason, like "Old Servers" and click on Confirm button.

  1. View Out Of Scope assets in the dedicated Out of Scope assets tab.

  1. Select the Out of Scope assets and bring them back in scope by clicking on Put back in scope button.
Note: Out of scope assets will be excluded from the TCO report and will only be available for viewing in the dedicated tab

Task 7. Generate Inventory and Performance data report

  1. In Google Cloud console, from the Navigation menu, select Migration Center and click on Create reports menu.

  1. Select Assets Details Export tile.

  1. In the opened page, you have several ways to export assets details, by using a group with several servers or by exporting all Servers that were previously collected using MC Discovery Client or imported using file import option.

  2. Click the Servers tab and select all servers, then click the Export to CSV/Google Sheets button above the filter field.

  1. At the popup window, click Export to Google Sheets to confirm and wait for the export to complete.

  2. When export completed the Open spreadsheet link will become available, click it in order to view the exported report in Google Sheets.

  3. In order to generate a Performance Data report, at the Create Reports window, select Performance data export tile.

  1. Similar to Assets Details Export, click the Export servers tab and select all servers, then click the Export to CSV/Google Sheets button above the filter field.

  2. At the opened popup window, keep the same number of Days per assets and click on the Export button again. Wait for report generation.

  1. At the popup window, click Export to Google Sheets to confirm.

  2. Wait for export to complete. Once export is completed the Open spreadsheet link will become available. Click it to view the exported report in Google Sheets.

Note: In this lab, performance data will be only available for assets collected using MC Discovery client.

Task 8. Generate TCO Report (Optional)

To generate a TCO report in the Migration Center, groups and migration preferences will have to be set.

  1. In Google Cloud, from the Navigation menu, select Migration Center and click on Groups sub-menu.

  2. Click + Create Group.

  1. Enter All Assets in the Group name field, Click Next.

  2. From the list select all assets by clicking the check box next to VM name label.

  3. Click the Create button.

  4. Repeat the steps above to create two more groups. Use filters to help selecting VMs.

    Group Name Select criteria
    windows-only All VMs running Windows OS
    linux-only All VMs running Linux OS

  5. From the Navigation menu, select Migration Center and click on Migration preferences sub-menu.

  6. Click + Create migration preferences button.

  7. Populate aggressive-optimization-3-year-commit preference name at the dedicated field.

  8. Expand Google Compute Engine section by clicking on it.

  1. Update Machine series with N2, N2D types.

  1. Expand Target Sizing Strategy section under Sizing and Network and choose Sizing Method Aggressive from the dropdown.

  1. Repeat the above steps to create one more preference set using following values:

    Preferences name moderate-optimization-1-year-commit
    Machine series C2, C2D
    Disk type SSD persistent disk
    Sizing method Moderate
    Product pricing track 1-year resource-based committed use discount (USD)

  2. From the Navigation menu, select Migration Center and click on Create Reports sub-menu. Select TCO and detailed pricing report tile.

  1. Enter lab-tco-report for report name, click Next.

  2. Select All-assets, windows-only and linux-only groups, click Next.

  1. Click the down arrow on the all-assets group to show assigned preferences.

  2. Select aggressive-optimization-3-year-commit from dropdown for preference 2.

  3. Select moderate-optimization-1-year-commit from dropdown for preference 3.

  1. Click Assign to all groups.

  2. Click Generate report.

  1. Report can take up to 5 min to generate. Green checkbox indicates that the report is ready to view.

  2. Click the report name to view details.

  3. To export a Detailed Pricing Report go to Export report option and select Export detailed pricing report to CSV/Google Sheets option.

  1. Once clicked, the report generation will start and after it's done, it will be possible to download generated CSV files to explore them locally and to export generated report to Google Sheets, by clicking on the Export to Google Sheets button.
NOTE: Both TCO and Detailed Pricing Report have an export to Google Slides and Google Sheets options correspondingly, however export to Google Slides feature is not available in the lab.

Task 9. Generate Network Dependencies report (Optional)

  1. In Google Cloud, from the Navigation menu, select Migration Center and click on Create Reports menu.

  1. Select Network Dependencies Export tile.

  2. Similar to Inventory and Performance data reports, you have an option to export a report for a group of assets.

  3. In the opened Groups tab, select all groups that were previously created at Task 8.

  4. Click on the Export to CSV button at the bottom of the screen.

  5. At the opened popup window, click Export button and wait for report generation.

  6. Click the Download button in order to download the generated Network Dependencies Report in CSV format.

  1. After the report file was downloaded to your local machine you can review the file by opening it using any text or csv file editor.
